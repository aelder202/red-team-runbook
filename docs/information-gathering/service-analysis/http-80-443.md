## Quick Service Fingerprint

The first thirty seconds on any HTTP(S) port:

```bash
curl -skI https://10.10.10.10                           # headers (server, x-powered-by)
whatweb http://10.10.10.10                              # tech stack fingerprint
nmap -p 80,443 -sV --script http-enum,http-title,http-methods,http-headers 10.10.10.10
```

For TLS inspection and cert SAN extraction (SAN entries often reveal other vhosts):

```bash
openssl s_client -connect 10.10.10.10:443 -showcerts </dev/null 2>/dev/null \
  | openssl x509 -noout -text | grep -E 'Subject:|DNS:'
```

!!! tip "Real-world"
    On external engagements, HTTP/HTTPS is almost always in scope and is usually the first (and sometimes only) foothold path. Fingerprint aggressively, check the cert SANs, run a quick `nuclei` templates pass, and move to manual Burp work against anything that looks non-standard.
